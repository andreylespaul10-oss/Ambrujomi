# CHAPTER 7
# ICONS
# Bliss Glow Design System
# Version 1.0
# ============================================================

# PURPOSE

The Icon System provides a consistent visual language for actions,
navigation and status indicators throughout the Bliss Glow platform.

Icons should improve usability by supporting text, not replacing it.

------------------------------------------------------------

# DESIGN PRINCIPLES

Icons must be:

• Simple

• Modern

• Consistent

• Recognisable

• Accessible

• Scalable

------------------------------------------------------------

# ACCESSIBILITY

Decorative icons

Marked as hidden from assistive technologies.

Functional icons

Require accessible labels.

Interactive icons must have a minimum touch target of

44 × 44 px.

------------------------------------------------------------

# CUSTOM ICONS

Custom icons should:

Follow the official stroke width

Use rounded corners

Align to the pixel grid

Maintain visual balance

Match the official icon family

------------------------------------------------------------

# ANIMATION

Allowed

Subtle rotation

Fade

Scale

Slide

Not Allowed

Excessive bouncing

Continuous spinning (unless loading)

Distracting effects

------------------------------------------------------------

# PERFORMANCE

Icons should be delivered as SVG whenever possible.

Avoid bitmap icons.

Optimise SVG paths.

Support tree shaking for icon libraries.

------------------------------------------------------------

# BEST PRACTICES

Always pair important icons with text.

Maintain consistent spacing.

Do not mix multiple icon styles.

Avoid unnecessary decorative icons.

Prioritise clarity over visual complexity.
