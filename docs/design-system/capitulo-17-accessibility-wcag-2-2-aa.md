# CHAPTER 17
# ACCESSIBILITY (WCAG 2.2 AA)
# Bliss Glow Design System
# Version 1.0
# ============================================================

# PURPOSE

Accessibility is a fundamental requirement for every Bliss Glow digital
experience.

All interfaces must be designed so that people with diverse abilities
can successfully browse products, complete purchases and manage their
accounts independently.

The minimum compliance target is WCAG 2.2 Level AA.

------------------------------------------------------------

# ACCESSIBILITY PRINCIPLES

Every interface should be:

• Perceivable

• Operable

• Understandable

• Robust

Accessibility must be considered during design, development and testing.

------------------------------------------------------------

# COLOUR CONTRAST

Normal Text

Minimum Ratio

4.5:1

Large Text

Minimum Ratio

3:1

Interactive Components

Must remain distinguishable in every state.

Colour alone must never communicate meaning.

------------------------------------------------------------

# KEYBOARD NAVIGATION

Every interactive component must support:

Tab Navigation

Shift + Tab

Arrow Keys (where appropriate)

Enter

Space

Escape

Visible keyboard focus is mandatory.

------------------------------------------------------------

# FOCUS MANAGEMENT

Focus indicators must:

Be highly visible

Remain consistent

Never be removed

Follow logical navigation order

Restore focus after closing dialogs or overlays

------------------------------------------------------------

# SCREEN READER SUPPORT

Every interactive element must provide:

Accessible Name

Accessible Description (where required)

Correct ARIA Roles

Meaningful Labels

Logical Reading Order

Decorative elements should be hidden from assistive technologies.

------------------------------------------------------------

# FORM ACCESSIBILITY

Every field must include:

Persistent Label

Associated Input

Error Identification

Helper Text

Validation Feedback

Required Field Indicator

Autocomplete Support

------------------------------------------------------------

# IMAGES

Decorative Images

Empty ALT attribute

Informative Images

Descriptive ALT text

Product Images

Meaningful product description

Charts

Text alternatives or data tables

------------------------------------------------------------

# LINKS

Links must:

Clearly describe their destination

Remain visually identifiable

Support keyboard focus

Avoid generic text such as:

Click Here

Read More

------------------------------------------------------------

# BUTTONS

Buttons must:

Clearly describe the action

Provide accessible labels

Remain keyboard accessible

Maintain adequate touch target size

Minimum

44 × 44 px

------------------------------------------------------------

# TABLES

Tables require:

Column Headers

Row Headers where applicable

Accessible captions

Logical reading order

Responsive alternatives on small screens

------------------------------------------------------------

# MULTIMEDIA

Videos should support:

Captions

Transcripts

Playback controls

Keyboard operation

Audio should never play automatically.

------------------------------------------------------------

# ANIMATION

Respect

prefers-reduced-motion

Disable non-essential animations when requested by the user.

Avoid flashing content.

Avoid motion that may trigger discomfort.

------------------------------------------------------------

# ERROR HANDLING

Errors should:

Explain the problem

Describe how to fix it

Appear near the affected field

Be announced to screen readers

Remain visible until resolved

------------------------------------------------------------

# LANGUAGE

The document language must be correctly declared.

Use British English consistently.

Avoid unnecessarily complex wording.

------------------------------------------------------------

# RESPONSIVE ACCESSIBILITY

Accessibility requirements apply equally across:

Desktop

Tablet

Mobile

Foldable Devices

High Zoom Levels

------------------------------------------------------------

# TESTING

Accessibility testing must include:

Keyboard-only navigation

Screen Reader Testing

Colour Contrast Analysis

Zoom Testing (up to 400%)

Reduced Motion Testing

Mobile Accessibility Testing

------------------------------------------------------------

# ASSISTIVE TECHNOLOGIES

Support

NVDA

JAWS

VoiceOver

TalkBack

Windows Magnifier

Browser Zoom

Speech Recognition Software

------------------------------------------------------------

# DESIGN TOKENS

focus.outline

focus.colour

colour.contrast.minimum

touch.target.minimum

motion.reduced

------------------------------------------------------------

# BEST PRACTICES

Accessibility should be integrated from the beginning of the project.

Do not treat accessibility as a post-development task.

Test frequently with automated and manual methods.

Include users with disabilities in usability testing whenever possible.
