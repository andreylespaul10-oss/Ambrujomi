# Bliss Glow Design System
# Version 1.0
# ============================================================

# PURPOSE

Cards are one of the primary building blocks of the Bliss Glow interface.
They organise content into visually distinct, reusable containers that
improve readability, hierarchy and interaction across the platform.

Cards must remain lightweight, responsive and consistent.

------------------------------------------------------------

# DESIGN PRINCIPLES

Cards should be:

• Clean

• Elegant

• Informative

• Interactive

• Responsive

• Accessible

------------------------------------------------------------

# INTERACTION RULES

Entire card may be clickable when representing a single destination.

Buttons inside cards take priority over card click events.

Interactive elements must provide hover and focus feedback.

------------------------------------------------------------

# ACCESSIBILITY

Cards must support

Keyboard Navigation

Visible Focus States

Screen Reader Labels

Accessible Contrast

Logical Reading Order

WCAG 2.2 AA Compliance

------------------------------------------------------------

# RESPONSIVE BEHAVIOUR

Cards expand to available width.

Images maintain aspect ratio.

Content wraps naturally.

Actions remain easily tappable on mobile.

------------------------------------------------------------

# PERFORMANCE

Use lazy loading for images.

Optimise image sizes.

Avoid excessive shadows.

Reuse card components across the application.
