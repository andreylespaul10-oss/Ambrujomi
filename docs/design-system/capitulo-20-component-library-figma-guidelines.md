# PURPOSE

This chapter defines how every reusable UI component should be organised,
documented and maintained across the Bliss Glow design ecosystem.

The objective is to ensure complete consistency between design,
development and future product evolution.

------------------------------------------------------------

# DESIGN PHILOSOPHY

Every component must be:

• Reusable

• Modular

• Scalable

• Accessible

• Responsive

• Fully documented

------------------------------------------------------------

# FIGMA BEST PRACTICES

Use Components instead of duplicated elements.

Use Variants instead of separate files.

Avoid detached instances.

Maintain a clean layer structure.

Use descriptive layer names.

Keep pages organised and versioned.
