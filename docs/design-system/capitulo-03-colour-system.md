# CHAPTER 3
# COLOUR SYSTEM
# Bliss Glow Design System
# Version 1.0
# ============================================================

# PURPOSE

The Colour System defines the official colour palette used throughout
the Bliss Glow ecosystem.

The palette is designed to communicate elegance, trust and modern
premium beauty while maintaining excellent readability and accessibility.

All colours must comply with WCAG 2.2 AA contrast requirements wherever
applicable.

------------------------------------------------------------

# DESIGN GOALS

• Premium appearance

• High readability

• Accessible contrast

• Consistent branding

• Scalable theme system

• Future dark mode support

# ACCESSIBILITY

Minimum contrast ratio

4.5:1 for normal text

3:1 for large text

Do not rely on colour alone to communicate information.

All semantic states must include icons or labels where appropriate.

------------------------------------------------------------

# GRADIENTS

Gradients should be subtle.

Primary Gradient

Rose → Soft Blush

Usage

Hero banners

Marketing sections

Never use gradients for body text.

------------------------------------------------------------

# FUTURE DARK MODE

The palette must support a future dark theme by mapping every colour to
an equivalent dark-mode token without changing component behaviour.
