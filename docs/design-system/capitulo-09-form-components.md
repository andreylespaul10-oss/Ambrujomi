# FORM COMPONENTS
# Bliss Glow Design System
# Version 1.0
# ============================================================

# PURPOSE

Form components allow customers and administrators to enter, edit and
submit information efficiently and accurately.

Every form must prioritise clarity, accessibility, validation and ease
of use while minimising user effort.

------------------------------------------------------------

# DESIGN PRINCIPLES

Forms should be:

• Simple

• Consistent

• Accessible

• Responsive

• Easy to complete

• Error tolerant

# LABELS

Position

Above the field

Alignment

Left

Font Size

14 px

Weight

Medium

Labels must always remain visible.

Floating labels are not recommended.

------------------------------------------------------------

# PLACEHOLDERS

Purpose

Provide examples only.

Placeholders must never replace labels.

Example

Email Address

Placeholder

name@example.com

# HELPER TEXT

Position

Below the input

Usage

Guidance

Validation hints

------------------------------------------------------------

# VALIDATION

Validation should occur:

During input where appropriate

On field exit

Before form submission

Errors should clearly explain how to resolve the issue.

------------------------------------------------------------

# ERROR MESSAGES

Display directly below the affected field.

Use:

Error icon

Error colour

Clear explanation

Example

------------------------------------------------------------

# RESPONSIVE BEHAVIOUR

Inputs expand to available width.

Touch targets remain at least

44 × 44 px.

Spacing follows the official spacing scale.

# BEST PRACTICES

Ask only for necessary information.

Group related fields logically.

Provide immediate validation feedback.

Avoid unnecessary mandatory fields.

Support browser autofill whenever possible.
