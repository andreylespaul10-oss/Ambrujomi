# CHAPTER 5
# SPACING SYSTEM
# Bliss Glow Design System
# Version 1.0
# ============================================================

# PURPOSE

The Spacing System establishes a consistent rhythm for layouts,
components and content throughout the Bliss Glow platform.

A unified spacing scale improves readability, usability, responsiveness
and development consistency.

------------------------------------------------------------

# DESIGN PRINCIPLES

Spacing should create:

• Visual balance

• Clear content hierarchy

• Comfortable reading

• Consistent layouts

• Predictable interfaces

• Premium appearance

Whitespace should be treated as an essential design element.

------------------------------------------------------------

# BASE UNIT

The design system follows an 8-point grid.

Base Unit

8 px

All spacing values should be multiples of 8 whenever possible.

Exceptions are permitted only for typography alignment or thin borders.

# RESPONSIVE ADAPTATION

Desktop

Full spacing scale

Tablet

Reduce large spacing by approximately 20%

Mobile

Reduce large spacing by approximately 30%

Small spacing values remain unchanged to preserve usability.

------------------------------------------------------------

# ACCESSIBILITY

Maintain sufficient spacing for touch interactions.

Minimum touch target

44 × 44 px

Interactive elements should never overlap.

Whitespace must improve readability rather than reduce information density.

------------------------------------------------------------

# BEST PRACTICES

Use the spacing scale consistently.

Avoid arbitrary values.

Keep related content visually grouped.

Separate unrelated content with larger spacing.

Do not rely on empty elements for spacing.
