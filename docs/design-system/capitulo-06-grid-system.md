# CHAPTER 6
# GRID SYSTEM
# Bliss Glow Design System
# Version 1.0
# ============================================================

# PURPOSE

The Grid System provides the structural foundation for every Bliss Glow
interface. It ensures consistent alignment, predictable layouts and a
responsive experience across desktop, tablet and mobile devices.

All pages and components must follow this grid unless a documented
exception has been approved.

------------------------------------------------------------

# DESIGN PRINCIPLES

The layout should always be:

• Structured

• Consistent

• Responsive

• Flexible

• Accessible

• Easy to scan

------------------------------------------------------------

# ALIGNMENT RULES

Use left alignment by default.

Maintain consistent vertical rhythm.

Avoid uneven spacing.

Align interactive elements whenever possible.

------------------------------------------------------------

# NESTED GRIDS

Nested grids are permitted for:

Cards

Dashboards

Checkout sections

Modal layouts

Nested grids must inherit spacing from the parent grid.

------------------------------------------------------------

# SAFE AREAS

Respect device safe areas for:

Mobile notches

Rounded screen corners

Gesture navigation areas

Floating interface elements

------------------------------------------------------------

# RESPONSIVE IMAGES

Images should:

Scale proportionally

Never stretch

Use responsive formats

Support lazy loading

Provide appropriate aspect ratios
