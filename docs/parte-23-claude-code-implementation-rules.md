# ============================================================
# PART 23 — CLAUDE CODE IMPLEMENTATION RULES
# Bliss Glow
# ============================================================

# IMPLEMENTATION OBJECTIVE

Claude Code must implement the Bliss Glow platform following this complete specification as the primary source of truth.

The goal is to create a premium, scalable, production-ready e-commerce platform with professional engineering standards.

The implementation must prioritize:

• Quality
• Maintainability
• Scalability
• Performance
• Security
• User Experience

------------------------------------------------------------

# GENERAL DEVELOPMENT RULES

Claude Code must:

• Do not simplify requested features without approval.
• Do not remove functionality to make development easier.
• Do not create temporary solutions unless clearly documented.
• Always consider future scalability.
• Follow professional software engineering practices.
• Write clean, readable, maintainable code.

------------------------------------------------------------

# PROJECT ARCHITECTURE RULES

The project should use:

• Modular architecture
• Reusable components
• Clear folder structure
• Separation of concerns
• Scalable patterns

Code should be organized by:

• Features
• Components
• Services
• Utilities
• API modules
• Database models

------------------------------------------------------------

# FRONTEND DEVELOPMENT RULES

The frontend must provide:

• Premium visual experience
• Responsive design
• Mobile-first approach
• Smooth interactions
• Accessible interfaces

Requirements:

• Reusable UI components
• Consistent design system
• Optimized rendering
• Clean state management

------------------------------------------------------------

# UI/UX RULES

The design must feel:

• Premium
• Modern
• Elegant
• Trustworthy
• Professional

Avoid:

• Generic templates
• Cluttered layouts
• Poor spacing
• Inconsistent styling

Every page should have a clear purpose.

------------------------------------------------------------

# COMPONENT DEVELOPMENT

Components must be:

• Reusable
• Independent
• Documented
• Easy to maintain

Examples:

• Product Cards
• Navigation
• Buttons
• Forms
• Modals
• Checkout Components
• Dashboard Widgets

------------------------------------------------------------

# CODE QUALITY RULES

Code must prioritize:

• Readability
• Performance
• Security
• Maintainability

Avoid:

• Duplicate code
• Hardcoded values
• Unnecessary complexity
• Poor naming conventions

------------------------------------------------------------

# DATABASE RULES

Database implementation must:

• Follow the defined architecture
• Maintain data integrity
• Use proper relationships
• Support future expansion

Before changing database structures:

• Analyze impact
• Update documentation
• Maintain compatibility

------------------------------------------------------------

# API DEVELOPMENT RULES

APIs must include:

• Clear endpoints
• Validation
• Authentication
• Error handling
• Documentation

Every API response should be predictable and consistent.

------------------------------------------------------------

# DROPSHIPPING IMPLEMENTATION RULES

For DSers/AliExpress:

Claude Code must create:

• Supplier integration layer
• Product synchronization system
• Order synchronization
• Tracking updates
• Error handling

The architecture must allow adding other suppliers later.

------------------------------------------------------------

# SECURITY RULES

Always implement:

• Secure authentication
• Protected routes
• Input validation
• Safe data handling
• Permission control

Never:

• Expose secrets
• Store sensitive information insecurely
• Trust user input without validation

------------------------------------------------------------

# PERFORMANCE RULES

The implementation must optimize:

• Page speed
• Images
• Database queries
• API responses
• Frontend rendering

Avoid unnecessary:

• Network requests
• Large bundles
• Heavy dependencies

------------------------------------------------------------

# SEO IMPLEMENTATION RULES

Every public page should include:

• Metadata
• Structured data
• Clean URLs
• Proper headings
• Optimized images

SEO must be implemented during development, not after launch.

------------------------------------------------------------

# TESTING RULES

Claude Code must create:

• Unit tests
• Integration tests
• Critical user flow tests

Critical flows:

• Product browsing
• Add to cart
• Checkout
• Payment
• Account creation
• Order tracking

------------------------------------------------------------

# DOCUMENTATION RULES

The project must include documentation for:

• Installation
• Configuration
• Architecture
• APIs
• Database
• Deployment
• Maintenance

------------------------------------------------------------

# DEVELOPMENT WORKFLOW

Follow this process:

1. Analyze requirements

2. Create implementation plan

3. Build architecture

4. Develop features

5. Test functionality

6. Review quality

7. Document changes

8. Prepare deployment

------------------------------------------------------------

# INTERACTIVE PROTOTYPE RULES

The prototype must include:

• Clickable navigation
• Realistic user flows
• Interactive components
• Responsive behavior
• Production-like experience

The prototype should not feel like static screens.

------------------------------------------------------------

# AI IMPLEMENTATION RULES

When implementing AI features:

• Protect customer privacy
• Allow human control
• Provide explainable results
• Avoid unnecessary automation

------------------------------------------------------------

# ERROR HANDLING RULES

Every feature must consider:

• User mistakes
• Network failures
• Payment failures
• Integration failures
• System errors

Errors should be helpful and actionable.

------------------------------------------------------------

# BEFORE COMPLETION CHECKLIST

Claude Code must verify:

✓ Features implemented

✓ Responsive design completed

✓ Tests passing

✓ Security reviewed

✓ Performance optimized

✓ Documentation updated

✓ Deployment prepared

------------------------------------------------------------

# FINAL DEVELOPMENT PRINCIPLE

Claude Code should approach Bliss Glow as if building a real commercial product for a global brand, not a simple demonstration website.

The final result should represent a premium e-commerce platform capable of competing with professional online stores.
