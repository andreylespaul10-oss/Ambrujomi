# ============================================================
# PART 10 — CUSTOMER ACCOUNT (MY ACCOUNT)
# Bliss Glow
# ============================================================

# ACCOUNT OBJECTIVE

The Customer Account is the customer's personal control center.

Its purpose is to allow customers to manage their profile, orders, addresses, payment methods, wishlist, notifications, and preferences in a secure and intuitive environment.

The experience should encourage repeat purchases and long-term customer loyalty.

------------------------------------------------------------

# ACCOUNT STRUCTURE

The customer dashboard should include:

1. Dashboard Overview
2. Personal Profile
3. Order History
4. Order Details
5. Track Orders
6. Saved Addresses
7. Payment Methods
8. Wishlist
9. Saved Shopping Cart
10. Notifications
11. Account Security
12. Communication Preferences
13. Logout

------------------------------------------------------------

# DASHBOARD

The first screen should display:

• Welcome message

• Customer name

• Recent orders

• Current order status

• Wishlist summary

• Personalized product recommendations

• Recently viewed products

• Available coupons

• Loyalty points (future feature)

• Quick action buttons

------------------------------------------------------------

# PROFILE

Allow customers to update:

• First Name

• Last Name

• Email

• Phone Number

• Date of Birth (optional)

• Profile Picture (optional)

Changes should be validated before saving.

------------------------------------------------------------

# PASSWORD MANAGEMENT

Support:

• Change Password

• Forgot Password

• Password Strength Indicator

• Two-Factor Authentication (future feature)

------------------------------------------------------------

# ADDRESS BOOK

Customers may:

• Add Address

• Edit Address

• Delete Address

• Set Default Address

Support multiple addresses.

------------------------------------------------------------

# PAYMENT METHODS

Allow customers to:

• View saved payment methods

• Add payment method

• Remove payment method

• Set default payment method

Sensitive payment data must never be exposed.

------------------------------------------------------------

# ORDER HISTORY

Display:

• Order Number

• Order Date

• Order Status

• Total Amount

• Payment Status

• Shipping Status

• Track Order Button

• View Details Button

------------------------------------------------------------

# ORDER DETAILS

Each order should display:

• Purchased products

• Product images

• Quantities

• Prices

• Shipping address

• Billing information

• Payment method

• Tracking information

• Invoice (if available)

------------------------------------------------------------

# ORDER TRACKING

Display:

• Current shipment status

• Estimated delivery

• Carrier

• Tracking code

• Shipping timeline

------------------------------------------------------------

# WISHLIST

Customers can:

• Add products

• Remove products

• Move products to cart

• Share wishlist (optional)

------------------------------------------------------------

# SAVED CARTS

Support:

• Save current cart

• Restore saved cart

• Rename saved cart (optional)

------------------------------------------------------------

# NOTIFICATIONS

Display:

• Order updates

• Shipping updates

• Promotions

• Account alerts

Customers should be able to mark notifications as read.

------------------------------------------------------------

# COMMUNICATION PREFERENCES

Allow customers to manage:

• Marketing emails

• SMS notifications

• Push notifications (future)

• Newsletter subscription

------------------------------------------------------------

# ACCOUNT SECURITY

Features:

• Change Password

• Active Sessions

• Recent Login Activity

• Two-Factor Authentication (future)

• Device Management (future)

------------------------------------------------------------

# DELETE ACCOUNT

Provide:

• Clear explanation

• Confirmation step

• Data retention notice

• Account recovery period (optional)

------------------------------------------------------------

# EMPTY STATES

Provide friendly screens when:

• No orders

• Empty wishlist

• No notifications

• No saved addresses

• No payment methods

Include buttons guiding users to continue shopping.

------------------------------------------------------------

# RESPONSIVE DESIGN

Desktop:

• Sidebar navigation

• Dashboard widgets

Tablet:

• Collapsible sidebar

Mobile:

• Bottom navigation or slide-out menu

• Large touch-friendly controls

------------------------------------------------------------

# ACCESSIBILITY

Support:

• Keyboard navigation

• Screen readers

• High contrast

• Focus indicators

• Accessible forms

------------------------------------------------------------

# PERFORMANCE

Requirements:

• Fast dashboard loading

• Optimized data fetching

• Lazy loading for large order history

• Smooth transitions

------------------------------------------------------------

# INTERACTIVE PROTOTYPE REQUIREMENTS

The interactive prototype must allow users to:

• Log in

• Register

• Edit profile

• Manage addresses

• Manage payment methods

• View orders

• Track orders

• View order details

• Add and remove wishlist items

• Manage saved carts

• Update preferences

• Log out

Every screen should behave like a production-ready customer portal.

------------------------------------------------------------

# BUSINESS GOALS

The Customer Account should:

• Increase customer retention

• Encourage repeat purchases

• Reduce support requests

• Build trust

• Personalize the shopping experience

------------------------------------------------------------

# DESIGN GOALS

The Customer Account should feel:

• Personal

• Secure

• Organized

• Elegant

• Easy to navigate

• Fast

• Mobile-first

• Premium

End of Part 10.
